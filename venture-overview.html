4 Critical Reflection on AI Tools

‚úÖ What Worked Well

  • Rapid prototyping: AI coding assistants accelerated development significantly
  • Personalization at scale: Claude provides unique responses for each user without scaling costs linearly
  • Natural language interface: Chat is intuitive ‚Äî no learning curve for users
  • Creative inspiration: AI suggested badge names, reward messages, and UI copy

‚ö†Ô∏è Challenges & Limitations

  • Hallucination risk: AI may provide incorrect travel information (visa rules, safety info)
  • Cost at scale: API costs per user could impact margins at high volume
  • Latency: AI responses take 1-3 seconds, affecting user experience
  • Context window limits: Can't include full travel history for power users
  • Dependency risk: Reliance on third-party API (Anthropic) for core feature

üîÆ Lessons Learned

  • Graceful degradation is essential: App must work without AI (demo mode saved us)
  • Prompt engineering is iterative: System prompts required multiple refinements
  • Hybrid approaches work best: Algorithm + AI is more robust than pure AI
  • User context is key: Generic AI is less valuable than context-aware AI

üöÄ Future Improvements

  • Fine-tune model on travel-specific data to reduce hallucinations
  • Implement response caching for common queries to reduce costs
  • Add streaming responses to improve perceived latency
  • Explore local/smaller models for cost-sensitive features
